Reference Data

            
Manage configurable tables used to enrich, classify, map, or suppress decisions.

          

          

            
            
            
          

        

        

          

            How reference data works
            
Reference data is a maintained table that rules can use without changing code. A rule chooses a table, matches one row by the match column, and returns a selected field from that row.

          

          

            

              Reference table
              product_catalog
              Used by rules to select a business table.
            

            

              Match column
              product_code
              Column matched against profile, context, or output values.
            

            

              Generated output
              =lookup("product_catalog", outputs.product_code, "priority")
            

            

              Optional validity
              valid_from / valid_to / ttl_seconds
              Use for offers, tariffs, bundles, or other expiring objects.
